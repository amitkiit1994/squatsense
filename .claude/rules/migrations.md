# Database Migration Rules (Alembic)

## Creating Migrations
```bash
alembic revision --autogenerate -m "description_in_snake_case"
```

## Applying Migrations
```bash
alembic upgrade head    # apply all pending
alembic downgrade -1    # rollback last migration
```

## Conventions
- Migration messages in snake_case: `add_email_verified_to_league_players`
- Always review autogenerated migrations — they may miss index changes or column type conversions
- PostgreSQL-specific types (UUID, JSONB) must have proper `import` in migration files
- The async driver is `postgresql+asyncpg` — alembic uses this via `run_async`
- Production migrations run automatically via `alembic upgrade head` in Dockerfile CMD before server start
